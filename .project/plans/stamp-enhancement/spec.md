# 印章增强

## 状态
`已记录，待开展`

## 技术方向
- **架构选择**：数据模型重构 —— 章配置从全局改为按页存储
- **技术栈**：Python + tkinter + Pillow + PyMuPDF
- **关键技术决策**：
  - 每页独立章配置：章的位置、大小、旋转、透明度均按 `(stamp_id, page_index)` 存储
  - 旋转使用 Pillow `Image.rotate()`，需处理旋转后尺寸变化和透明背景
  - 页面方向自适应：根据页面宽高比判断横/纵，印章参数（如大小比例）可分别配置

## 功能需求

### 2. 旋转支持
- `StampData` 或页级配置增加 `rotation: float = 0.0` 字段（范围 0~360）
- 控制面板增加旋转角度输入/滑块
- 预览渲染时：对印章应用旋转
- 导出时：所有处理器应用相同旋转逻辑
- 旋转中心为印章中心，保持位置不变

### 3. 纵 + 横页面公章支持
- 页面加载时识别方向：宽高比 > 1 为横页，< 1 为纵页
- 印章大小比例可按页面方向分别配置（或自动适配）
- 预览和导出时，印章尺寸基于当前页的实际尺寸计算
- 混合方向文档（如 PDF 中既有横页又有纵页）每页独立处理

### 6. 单独每页设置章
- 替换原有「所有页共享同一套章配置」逻辑
- 数据模型：章配置按 `(stamp_id, page_index)` 存储
- UI 行为：
  - 切换预览页面时，印章的位置/大小/旋转/透明度跟着变
  - 当前页的章配置修改只影响当前页
  - 提供「应用到所有页」快捷操作（可选）
- 导出时：每页按自己的配置盖章

## 非功能需求
- 性能：无特殊要求
- 安全：无特殊要求
- 数据：章配置 schema 变更，不保留旧数据兼容
- 兼容性：无特殊要求
