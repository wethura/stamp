# 体验优化

## 技术方向
- **架构选择**：现有三层架构上增量修改，无架构变更
- **技术栈**：Python + tkinter + Pillow，无新依赖
- **关键技术决策**：
  - 透明度使用 Pillow alpha 通道乘法实现，预览和导出统一处理
  - 拖放使用 tkinter 原生事件绑定，macOS 用 `<Drop>`，Windows/Linux 用 `<<Drop>>`
  - 删除按钮改为显式布局（pack/grid），替代可能被遮挡的 place 定位

## 功能需求

### 5. 删除章按钮显示
当前 `controls_panel.py` 已有删除按钮（红色 ✕），但使用 `place` 绝对定位在缩略图右上角，可能被遮挡或不易点击。

改为：
- 在章列表项底部添加显式的「删除」文字按钮
- 点击后弹出确认对话框，确认后删除
- 删除后同步更新选中状态和编辑状态

### 7. 拖入文档支持
- 绑定主窗口的拖放事件（macOS `<Drop>` / Windows & Linux `<<Drop>>`）
- 拖入文件时：
  1. 提取文件路径
  2. 调用 `HandlerRegistry.get_handler()` 判断格式是否支持
  3. 支持则直接加载，不支持则状态栏提示「不支持的文件格式」
- 支持拖入单个文件（暂不支持多文件同时拖入）

### 1. 透明度效果更好
当前 `stamp.py` 只有去白底逻辑，无透明度控制。

新增：
- `StampData` 增加 `opacity: float = 1.0` 字段（范围 0.0~1.0）
- 控制面板「编辑章」区域增加透明度滑块（0%~100%，默认 100%）
- 预览渲染时：对印章图片应用 alpha 通道乘法
- 导出时：PDF/Image/Excel 处理器均应用相同透明度逻辑
- 实现方式：Pillow `Image.putalpha()` 或 numpy alpha 乘法

## 非功能需求
- 性能：无特殊要求
- 安全：拖入文件路径仅用于打开，不做执行或解析外部内容
- 数据：无特殊要求
- 兼容性：macOS / Windows 拖放事件行为差异需在两种平台验证
